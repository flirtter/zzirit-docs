# 세션7 회고 — 번개탭 프로필 카드 리그레션

## 무엇이 깨졌나

번개탭에서 마커 클릭 시 뜨는 프로필 카드 피드가 정상 동작하지 않음:
1. **카드 드래깅 불가** — 하단 다른 유저 카드로 스와이프 안 됨
2. **프로필 상세 진입 불가** — 카드 탭 시 상세 화면으로 이동 안 됨
3. **찌릿/채팅 버튼 디자인 다름** — Figma 기준과 불일치
4. **ReportBottomSheet 제거** — 신고 기능 소실

## 근본 원인

커밋 `8ab5b5b` ("번개탭 마커 이미지 근본 수정 + 데모 데이터 전면 제거")에서 **마커 수정 범위를 벗어나** `LightningProfileCard.tsx`의 레이아웃을 함께 변경함.

### 구체적 변경과 영향

| 변경 | 이전 (작동) | 이후 (깨짐) | 영향 |
|------|------------|------------|------|
| `pagingEnabled` | ✅ 있음 | ❌ 제거 | FlatList 스냅/페이징 불가 → 카드 드래깅 안 됨 |
| 카드 높이 | `SCREEN_HEIGHT` | `SCREEN_HEIGHT - 150` | 카드 짤림, 비율 깨짐 |
| 배경색 | `#FFF` | `transparent` | 뒤에 지도 비침, 카드 가독성 저하 |
| `ReportBottomSheet` | 포함 | 삭제 | 신고 기능 소실 |
| `BottomTabBar` | 없음 | 추가 | 프로필 카드 화면에 불필요한 탭바 |

### 이전 작동 버전

```
git show 6aff0aa:components/lightning/LightningProfileCard.tsx
```

## 왜 발생했나

1. **범위 초과 변경** — "마커 이미지 수정"이라는 목적에 "프로필 카드 레이아웃 리팩토링"을 같은 커밋에 포함. 리뷰 없이 머지됨.
2. **시각적 QA 미실시** — 마커가 보이는지만 확인하고, 마커 클릭 후 프로필 카드 동작은 테스트하지 않음.
3. **Simplify 과잉** — "죽은 코드 제거" 명목으로 ReportBottomSheet를 삭제했으나, 이는 사용 중인 기능이었음.

## 교훈 (다음 세션부터 적용)

### 1. 한 커밋 = 한 목적
마커 수정 커밋에 카드 레이아웃 변경을 포함하지 않는다.
관련 없는 변경은 별도 커밋으로 분리.

### 2. 변경 후 해당 화면 전체 플로우 테스트
파일을 수정했으면, 그 파일이 사용되는 **전체 사용자 플로우**를 테스트한다:
- 마커 수정 → 마커 보이는지 + **마커 클릭 후 프로필 카드** + **카드 드래깅** + **상세 진입**
- 프로필 카드 수정 → 카드 렌더링 + 버튼 동작 + 네비게이션

### 3. "죽은 코드"인지 확인 후 삭제
`ReportBottomSheet`처럼 현재 import되고 있는 컴포넌트는 죽은 코드가 아니다.
삭제 전 `grep -r "ComponentName"` 으로 사용처 확인.

### 4. 리그레션 체크리스트
큰 변경(10+ 파일) 후 아래 항목 확인:
- [ ] 번개탭: 지도 로딩 → 마커 표시 → 마커 클릭 → 프로필 카드 → 드래깅 → 상세 진입
- [ ] 좋아요: 받은/보낸/ZZIRIT 탭 전환 → 카드 표시 → 블러/결제
- [ ] 채팅: 채팅 목록 → 채팅방 진입 → 메시지 표시
- [ ] MY: 프로필 이미지 → 이름 → 볼트 → Like 섹션
- [ ] 미팅: 목록 → 상세 → 작성 → 작성자 이름

## 수정 방향

`LightningProfileCard.tsx`를 커밋 `6aff0aa` 버전으로 복원하되, 마커 관련 변경은 유지:
- `pagingEnabled` 복원
- 카드 높이 `SCREEN_HEIGHT` 복원
- 배경색 `#FFF` 복원
- `ReportBottomSheet` 복원
- 불필요한 `BottomTabBar` 제거
