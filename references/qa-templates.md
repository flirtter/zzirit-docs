# QA 템플릿 & TC 생성 가이드

## 1. QA 이슈 리포트 템플릿

### QA팀 표준 양식

```markdown
---
surface: {{surface}}
severity: critical | high | medium | low
status: open | fixing | resolved
date: {{YYYY-MM-DD}}
reporter: {{name}}
---

## 테스트 환경
- **앱 버전**:
- **OS/기기**:
- **테스트 URL**:

## 테스트 데이터
- **사용자번호**:
- **테스트 계정**:

## 재현절차
(재현빈도, 특정 계정 여부 포함)

1.
2.
3.

## 오류내용



## 기대결과
(기획 내용, REAL 확인)



## 참고
(추가 정보, 관련 이슈 링크 등)



## TC
(관련 테스트 케이스 번호)

```

### 디자인팀 QA 양식 (간결 버전)

디자인 파리티 이슈에 적합한 간결한 형식:

```markdown
## <수행절차>
홈 > 폴더 > 폴더 편집

## <오류내용>
편집뷰와 전체보기 영역이 비슷한데 편집뷰의 경우 간격이 좌우 24로 넓어보임

## <기대결과>
전체보기에 맞춰 간격조정 부탁드립니다. (맞는 간격은 16dp정도로 보임)

## <참고 이미지>
AS IS: ![[as-is.png]]
TO BE: ![[to-be.png]]
```

## 2. 이슈 심각도 기준

| 심각도 | 기준 | 예시 |
|--------|------|------|
| **critical** | 앱 크래시, 데이터 손실, 보안 이슈 | 채팅 전송 시 앱 종료, 결제 오류 |
| **high** | 핵심 기능 동작 불가 | 카메라 에러, 위치 전송 안됨, 로그인 실패 |
| **medium** | 기능은 동작하나 디자인/UX 불일치 | 버튼 위치 다름, 색상 불일치, 간격 차이 |
| **low** | 사소한 UI 이슈, 개선 사항 | 텍스트 오타, 미세 정렬, 폰트 크기 |

## 3. QA 이슈 작성 위치

```
02. Project/{surface}/QA/{surface}-qa-{YYYYMMDD}.md
02. Project/{surface}/attachments/  ← 스크린샷
```

- 이슈 파일은 surface별 QA 폴더에 생성
- 스크린샷은 해당 surface의 attachments 폴더에 저장
- QA.md (대시보드)에 링크 추가
